# Oviya v4 - Phase 1: Brand + dual-mode design system + assets

Foundation phase. Brand, the dual-mode (light + dark) OKLCH design system, the `/design` showcase
(brand bible), the premium dual-mode UI primitives, and asset generation. The studio, engine, tabs,
video, and landing were not touched. Internal "drape" identifiers (repo, `drape_` DB namespace,
buckets, env vars, deploy names) are unchanged.

## 1. The accent decision: OXBLOOD
Committed to oxblood over deep forest green. It reads as both Western luxury (lacquer, gallery wall,
wine) and India-rooted richness (maroon textile) without ethnic cliche; it holds full chroma on warm
cream and shifts cleanly lighter/more saturated on near-black; and it carries the witty, confident,
art-house register better than a safer green, which trends quiet-luxury-SaaS. Light accent
`#912530` (oklch 0.44 0.142 20), dark accent `#cf5d61` (oklch 0.62 0.145 20). Indigo, antique gold
(hairline/stipple only), and sandstone are rationed support intelligence.

## 2. Type
Replaced the templated Fraunces/Inter. Self-hosted via `next/font` (swap licensed kits in later by
changing one variable):
- Display: **Bodoni Moda** (high-contrast editorial Didone, the GT-Sectra/Canela register).
- Body/UI grotesk: **Hanken Grotesk** (premium neutral, Satoshi-adjacent).
- Mono: **DM Mono** (atelier process + technical metadata).
Documented as tokens: three font variables, a ~1.25 modular scale (`--step--1` .. `--step-6`),
tracking and line-height tokens.

## 3. Token architecture (OKLCH, 3 tiers)
Generated by `scripts/build-tokens.mjs` (culori) into `src/app/tokens.css` + `src/app/_tokens.json`:
- **Primitives:** warm neutral spine (14 steps, cream to warm onyx) + oxblood accent ramp (10) +
  support hues, authored in **OKLCH** with an `@supports not (color: oklch())` **sRGB hex fallback**.
- **Semantics (theme-switched):** `--surface-base/raised/overlay/sunken`, `--text-primary/secondary/
  muted`, `--border-subtle/strong`, `--accent-default/hover/contrast`, `--glow-accent`, etc. Emitted
  under `:root`/`.light` (light) and `.dark` (dark); re-declared per scope so the `/design`
  side-by-side panes theme correctly.
- **Components + legacy aliases:** `--button-bg`, `--card-bg`, ... plus every v1-v3 token name
  (`--ink`, `--accent`, `--surface`, `--fog`, ...) mapped onto the new semantics, so the existing
  studio/landing inherit the new system unchanged.
- Theming via `next-themes` (class strategy, persisted, default dark so nothing regresses).
- **WCAG 2.2 verified on real values** by `scripts/check-contrast.mjs` (runs in the gate): body
  16.5:1 light / 17.1:1 dark, secondary 6.3 / 10.0, text-on-accent 7.7 / 5.2, accent-as-UI 7.1 /
  5.2, all >= threshold in both themes.

## 4. Primitives (all dual-mode, semantic-token-based)
`ThemeToggle`, `Button` (primary accent-fill / secondary hairline / ghost, press, hover glow bloom,
inline loading), `Slider` (semantic labeled ends), `Sheet`/`Drawer`, `Modal`, `ThumbTile` +
`GalleryPicker` (no native dropdowns), `Toast`, `AtelierLoader` (staged narrated craft steps in
mono), `RequestControl` (the reusable "Can't find what you want?" escape hatch), `AliveBackground`
(token-driven gradient scaffold; WebGL deferred to the motion phase), film grain (theme-aware),
refined `Wordmark`/`Monogram` (antique-gold ring + a single vertical hairline nodding to Indic-script
verticality). Brand voice baked into defaults ("Hand us the garment. We'll handle the drama.",
"Styling the shoot", "Your editorial is ready for its close-up.").

## 5. The `/design` showcase (brand bible)
`/design` renders the whole system: live theme toggle, light + dark palette panes side by side,
primitive ramps, the type scale and three voices, every primitive in every state, motion tokens,
the wordmark, the atelier loader, the alive-background scaffold, voice samples, and the asset
gallery with per-image model tags + the comparison.

## 6. Assets (mixed-model, tagged)
`scripts/generate-assets-v4.ts` -> `public/v4/**` + `public/v4/manifest.json` +
`docs/ASSET_PROMPTS_V4.md` (full prompt + model + cost table and the comparison write-up). Saved
separately from the v3 set so nothing in the current app changes.
- **35 v4 assets** generated (3 heroes, 14 gallery spanning the luxury-style vocabulary with
  international + Indian 50/50 and diverse ethnicities/skin tones/ages, 10 Looks, 3 empty/onboarding,
  5 texture plates).
- **Model reality:** the Gemini key was credit-depleted at run start, so v4 completed on **fal Flux**
  (cheap, and genuinely strong, verified by eye: oxblood chiaroscuro hero, warm heritage saree, clean
  plaster plate). The v3 **Gemini Nano Banana Pro** set (25 images, incl. the verified before/after
  pairs) is surfaced on `/design` as the Gemini side of the comparison.
- **Actual spend: ~$0.58** (35 fal images at ~$0.025; Gemini ~$0.00, depleted). Far under the
  $20-45 budget. See the doc's comparison table and recommendation.
